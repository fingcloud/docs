# دستورات

## بررسی اجمالی

از دستور `fing` برای استقرار برنامه‌ها استفاده می شود. برای اجرای آن یا باید در مسیر فعلی  پروژه باشید و یا مسیر پروژه مورد نظر را به آن ارائه دهید.

###### استفاده پایه‌ای
اجرای دستور `fing` در مسیر فعلی پروژه:

```
$ fing up
```




###### استفاده طولانی‌تر
اجرای دستور `fing` با اضافه کردن مسیر پروژه:
```
$ fing --path [path/to/project] deploy
```


## انتخاب پلتفرم
هنگام ایجاد یک برنامه جدید، فینگ CLI  به طور خودکار پلتفرمی که از آن استفاده می کنید را شناسایی می کند و بر این اساس تنظیمات پیش فرض برنامه را ارائه می دهد. به عنوان مثال اگر برنامه شما  `NodeJs` باشد، فینگ CLI به صورت خودکار آن را پلتفرم `node` تشخیص  می‌دهد و عملیات استقرار را آغاز می کند. البته شما می توانید هنگام اجرای دستور `fing up` پلتفرم برنامه را هم مشخص کنید:
```
$ fing up --platform=node
```
